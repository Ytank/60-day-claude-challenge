      Recruiter's Orders
      How this was built
      System prompt design

      The assistant is scoped narrowly to one job: comparing a job description against a resume and returning a strict JSON verdict. It's told to adopt a blunt, expert-recruiter voice in the text fields only — the JSON schema itself stays fixed so the UI can render reliably. It's instructed to ask for nothing back conversationally (single-shot), to handle missing/garbled input by lowering confidence rather than guessing wildly, and to refuse politely (via a low score + explanation) if the pasted content isn't actually a job description or resume.

      Why the UI looks this way

      A "case file" aesthetic (manila paper, typewriter font, a rubber verdict stamp) matches the blunt-recruiter tone better than a generic chat box — it frames the tool as a dossier review, not small talk. The circular dial gives an instant read on fit; the stamp delivers the verdict with the bluntness the persona promises; keyword tags make the ATS-style matching tangible.
